# 產品需求文件 (PRD) - 個人記帳簿系統

## 1. 專案概述
- **背景與動機**：
  現代人常因為無法掌握金錢流向而造成超支。本系統旨在提供一個簡單明瞭的工具，讓使用者可以清楚記錄每天所花費的金額、項目以及收入來源，進一步幫助使用者培養良好的理財習慣並掌握財務狀況。
- **目標用戶**：
  任何想要記帳、希望清楚掌握自身財務狀況的人。
- **核心價值主張**：
  提供最直覺、無負擔的介面記錄與分類每日的收支，並能即時自動統計最終餘額，讓記帳變為輕鬆的日常小事。

## 2. 功能需求
系統涵蓋的主要功能如下：

1. **記錄支出項目（花費在什麼上）**
   - **使用者故事**：作為記帳用戶，我希望可以輸入並儲存我花費的具體項目（例如：午餐、交通等），以便知道錢花去了哪裡。
2. **記錄支出金額**
   - **使用者故事**：作為記帳用戶，我希望可以針對各項花費輸入對應的金錢數值，以便精準計算我的總花費。
3. **記錄收入金額**
   - **使用者故事**：作為記帳用戶，我希望可以輸入每天或每個月獲得的實質收入金額，以便了解我擁有的總資金。
4. **紀錄收入項目**
   - **使用者故事**：作為記帳用戶，我希望可以註記這筆收入的來源項目（例如：薪水、投資、獎金等），以便追蹤主要資金來源。
5. **最終統計餘額**
   - **使用者故事**：作為記帳用戶，我希望系統能在首頁自動顯示總收入減去總支出的最終餘額，以便我隨時檢視目前的財務健康狀況。

## 3. 非功能需求
- **技術限制**：
  - 伺服器端：Python 搭配 Flask 框架
  - 頁面渲染：Jinja2 模板引擊
  - 資料庫：SQLite (適合輕量級應用與單一檔案儲存)
- **效能與安全考量**：
  - **效能**：系統操作包含資料讀寫與餘額計算，應能在合理範圍內快速響應（例如 1 秒以內載入完畢）。
  - **安全性**：防止基本的 SQL Injection (應使用參數化查詢或 ORM) 和 Cross-Site Scripting (XSS，利用 Jinja2 預設開啟 Autoescaping)。

## 4. MVP 範圍 (最小可行性產品)
- **Must Have（必須具備）**：
  - 建立新的收入/支出紀錄（包含項目、金額、日期與分類）。
  - 在頁面上條列所有過往紀錄，包含歷史收入與支出明細。
  - 即時計算總收入、總支出以及最終結餘，並展示在顯眼位置。
- **Should Have（應該具備）**：
  - 紀錄的編輯與刪除功能（打錯字可以修正）。
  - 資料能依日期排序。
- **Nice to Have（可以具備）**：
  - 收支圖表分析（圓餅圖或長條圖）讓視覺化更加精美。
  - 多使用者帳號登入系統，能讓每個使用者有自己獨立的記帳空間。
